## 한 일
- `contractType` 1차 구현을 완료했다. 모바일은 `/select` 검토 화면에서 `제조·건설·서비스` / `농축산·어업`을 선택해 session과 multipart payload로 전달하고, 서버는 `contractType`을 수신·정규화한다.
- 서버 프롬프트는 유형별 체크리스트를 분기한다. 제조·건설·서비스는 교대제, 월 통상임금, 연장·야간·휴일근로 수당을 더 강하게 보고, 농축산·어업은 농번기·성어기와 근로시간·휴게·휴일 단정 금지를 명시한다.
- 제목 mismatch 안내는 법률 판정이 아니라 "선택한 종류와 다를 수 있음" 안내로 프롬프트에 넣었다. 응답 스키마와 저장 기록 모델은 바꾸지 않았다.
- 검증: `python3 -m py_compile main.py ai_service.py models.py`, `python3 -m unittest tests.test_ai_service_validation -v`, `npx tsc --noEmit`, `git diff --check` 통과.
