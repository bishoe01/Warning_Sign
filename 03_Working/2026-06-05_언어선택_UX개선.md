# 2026-06-05 언어 선택 UX 개선

## 문제

기존 언어 선택 UI는 16개 언어를 가로 스크롤 탭으로 보여줬다. 한 화면에 전체 선택지가 들어오지 않고, 사용자가 옆으로 더 넘겨야 한다는 사실도 명확하지 않았다. 언어 선택은 앱을 읽기 위한 첫 관문이므로, 일부 선택지가 숨겨지는 탭 구조는 맞지 않는다.

또한 각 항목에 `nativeName`과 `englishName`을 함께 보여주면 영어를 잘 안다는 전제가 생긴다. 이 앱의 대상 사용자는 영어 사용자가 아니라 "자신이 읽을 수 있는 언어를 찾는 사용자"이므로, 화면에는 언어명을 해당 언어 자체로 보여주는 것이 더 안전하다.

## 결정

- 설정 화면: 16개 언어를 2열 그리드로 보여준다.
- 결과 화면: 결과 본문을 밀어내지 않도록 현재 선택 언어만 버튼으로 보여주고, 누르면 언어 선택 모달을 연다.
- 언어 항목에는 `nativeName`만 표시한다.
- 선택 상태는 체크 아이콘과 파란 강조로 표시한다.
- `englishName`은 코드 내부 메타데이터로만 남기고 화면에는 노출하지 않는다.
- 국기 아이콘은 쓰지 않는다. 언어와 국가는 1:1이 아니기 때문이다.
