# PRD: AI 근로계약서 도우미 MVP

> 상태: DRAFT  
> 기준일: 2026-05-31  
> 갱신: 2026-06-01 — UI 리디자인(Toss·인앱 카메라) 반영. 저장·재열람과 값 포함 다국어를 다음 단계로 추가.  
> 기준 문서: `01_Idea/idea_.md`, `01_Idea/2026-05-31_idea_review.md`

## 1. 제품 한 줄 설명

외국인 근로자가 한국어 근로계약서를 촬영하면, OCR과 AI가 핵심 근로조건과 확인이 필요한 조항을 쉬운 한국어와 모국어로 설명해 주는 모바일 앱 MVP.

## 2. 문제

외국인 근로자는 한국어 근로계약서를 받았을 때 임금, 근로시간, 휴일, 계약기간, 공제, 수습기간, 위약금 같은 중요한 내용을 정확히 이해하기 어렵다. 일반 번역기는 문장을 바꿔줄 수는 있지만, 어떤 항목을 특히 확인해야 하는지, 사장님에게 무엇을 물어봐야 하는지까지는 알려주기 어렵다.

## 3. 대상 사용자

### 1차 사용자

한국어 근로계약서를 혼자 확인해야 하는 외국인 근로자.

### 사용 상황

- 채용 직후 근로계약서를 받은 순간
- 계약서에 서명하기 전
- 임금, 근무시간, 공제, 수습기간 같은 조건을 다시 확인하고 싶은 순간

## 4. 목표

- 계약서 이미지를 실제 OCR로 텍스트화한다.
- AI가 근로조건과 주의 조항을 구조화된 JSON으로 만든다.
- 사용자가 핵심 조건을 카드 형태로 빠르게 확인한다.
- 쉬운 한국어, 영어, 베트남어 설명을 전환해 볼 수 있다.
- 결과가 법률 자문이 아니라 이해 보조임을 명확히 알린다.

다음 단계(2026-06-01 추가):

- 분석 결과를 기기에 저장해 나중에 다시 열어볼 수 있다(1회성 탈피).
- 사용자가 언어 토글/리스트에서 읽고 싶은 언어를 고르면 앱 전체 UI와 핵심 조건·주의 조항을 **값까지** 해당 언어로 본다.

## 5. 비목표

- 법률 위반 여부 확정 판단
- 계약 무효 판단
- 노무 상담 대체
- 회원가입·로그인, 서버측 계정/DB
- 관리자 페이지
- iOS 배포
- 모든 계약서 유형 지원

> 변경(2026-06-01): 기존 비목표였던 '계약서 저장함'은 **기기 로컬 저장(계정 없음)** 한정으로 범위에 넣는다. 서버에는 저장하지 않는다는 §9 안전 요구사항은 그대로 유지한다. → F-12·F-13

## 6. 핵심 사용자 흐름

```text
홈 화면
→ 계약서 촬영 또는 이미지 선택 (여러 장 가능)
→ 분석 요청
→ 업로드 중/글자 읽는 중/AI 분석 중 상태 표시
→ 결과 화면
→ 핵심 근로조건 카드 확인
→ 주의 조항 카드 확인
→ 언어 전환
→ 참고용 안내 확인
→ (분석 직후) 기기에 자동 저장
→ 홈의 "내 기록"에서 과거 분석 재열람 · 삭제
```

## 7. 기능 요구사항

| ID | 기능 | 우선순위 | 완료 기준 |
|---|---|---:|---|
| F-01 | 홈 화면 | P0 | 앱 실행 시 서비스 이름과 분석 시작 버튼이 보인다. |
| F-02 | 이미지 선택 | P0 | 갤러리에서 계약서 이미지를 선택하고 미리 볼 수 있다. |
| F-03 | 카메라 촬영 | P1 | 카메라로 계약서 사진을 찍고 미리 볼 수 있다. |
| F-04 | 이미지 업로드 | P0 | 앱이 이미지를 FastAPI 서버로 전송한다. |
| F-05 | OCR 텍스트 추출 | P0 | 서버가 OCR API로 텍스트를 추출하고 반환한다. |
| F-06 | AI 분석 | P0 | 서버가 OCR 텍스트를 OpenAI API로 분석해 정해진 JSON을 반환한다. |
| F-07 | 결과 카드 | P0 | 앱이 핵심 근로조건과 확인이 필요한 조항을 카드로 표시한다. |
| F-08 | 다국어 설명 | P1 | 쉬운 한국어, 영어, 베트남어 설명을 전환할 수 있다. |
| F-09 | 분석 상태 화면 | P0 | 업로드, OCR, AI 분석 단계를 사용자가 볼 수 있다. |
| F-10 | 법률 자문 아님 고지 | P0 | 결과 화면에 참고용 안내가 항상 보인다. |
| F-11 | 샘플 모드 | P1 | API 장애 시 샘플 계약서 결과로 데모를 이어갈 수 있다. |
| F-12 | 분석 결과 로컬 저장 | P2 | 분석이 끝나면 결과(요약·주의조항·계약서 이미지들·시각)를 기기에 저장한다. |
| F-13 | 저장 내역 열람 | P2 | 홈에서 과거 분석을 목록으로 보고 다시 연다(마이페이지는 P9). |
| F-14 | 값 포함 다국어 | P2 | 항목 라벨뿐 아니라 요약 값·주의 조항 내용도 선택 언어로 본다. |
| F-15 | 마이페이지(언어 설정) | P2 | 사용자가 언어 토글/리스트에서 앱 언어를 고르고, 선택값이 앱 전체 UI와 분석 결과에 함께 적용되며 기기에 저장된다. |
| F-16 | 멀티페이지 분석 | P2 | 여러 장 계약서를 촬영/선택해 한 번에 업로드하고, 서버가 장별 OCR 후 합쳐 분석한다. |
| F-17 | 사진 출처 하이라이트 | P3 | 주의 조항 카드에서 `사진에서 확인`으로 실제 계약서 이미지 위치를 하이라이트하고, 매칭 실패 시 원문 인용/낮은 신뢰도 안내를 제공한다(후속 단계, Phase 10). |

## 8. API 요구사항

### `GET /health`

서버 정상 작동 확인.

```json
{
  "ok": true
}
```

### `POST /analyze-contract`

계약서 이미지를 받아 OCR과 AI 분석 결과를 반환한다.

요청:

```http
Content-Type: multipart/form-data

files: contract_image_1.jpg, contract_image_2.jpg, …   (1장 이상 N장; F-16 멀티페이지)
language: ko | en | ne | tet | ru | mn | my | bn | vi | uz | id | zh | km | ky | th | lo
```

## 9. 안전 요구사항

- API 키는 앱에 넣지 않고 서버 환경변수로 관리한다.
- 계약서 이미지와 OCR 텍스트를 서버에 저장하지 않는다.
- 서버 로그에 계약서 원문을 남기지 않는다.
- 업로드 전 "분석을 위해 계약서 이미지가 서버로 전송된다"는 안내를 보여준다.
- AI 응답은 JSON 스키마로 검증하고, 실패하면 사용자에게 재시도 안내를 보여준다.
- "불법", "무효", "확정", "위험 판정" 같은 단정 표현을 피한다.

> 💡 **어필 포인트(공모전):** "계약서 이미지·분석 결과를 서버에 저장하지 않고 **기기 로컬에만** 둔다"는 점은 민감문서(임금·신상)를 다루는 앱의 **프라이버시 차별점**이다. 발표자료·신청서에서 신뢰 포인트로 강조한다. 분석 기록 저장(F-12·F-13)도 같은 원칙으로 **로컬 전용**(계정·서버 DB 없음), 백엔드는 OCR/AI 프록시 역할만 유지.

## 10. 성공 기준

- Android 기기 또는 에뮬레이터에서 앱이 실행된다.
- 샘플 계약서 이미지로 분석 흐름을 끝까지 시연할 수 있다.
- 실제 OCR API가 최소 1회 이상 정상 동작한다.
- AI 분석 결과가 앱 카드 UI에 표시된다.
- 발표용 데모 영상에서 촬영/선택, 분석, 결과 확인 흐름이 1분 안에 보인다.
