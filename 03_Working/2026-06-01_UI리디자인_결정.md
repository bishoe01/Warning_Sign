# 2026-06-01 UI/UX 리디자인 결정 및 보고서 어필포인트

> 한 줄 결정: **앱 UI를 "임시 화면" 수준에서 벗어나 Toss(토스) 스타일의 모던 모바일 디자인으로 통일하고, 핵심 기능인 '계약서 촬영'을 인앱 스캐너(실시간 뷰파인더)로 구현한다.**

이 문서는 (1) 디자인 결정 기록과 (2) 신청서/보고서에 그대로 옮길 수 있는 어필포인트를 함께 남긴다. 코드 구현 전 단계의 합의안이다.

---

## 1. 배경 — 왜 고쳤나

기존 앱 화면(`04_App/mobile`)의 문제를 점검한 결과:

- 이모지(📄 🖼️)가 핵심 비주얼이라 "앱"이 아니라 "프로토타입"처럼 보였다.
- **카메라 촬영 기능이 없었다.** 갤러리에서 이미지 불러오기(`launchImageLibraryAsync`)만 있었음 — "촬영하면 분석한다"는 제품 전제와 맞지 않았다.
- SafeArea 미적용(노치 영역 깨짐), 디자인 토큰·아이콘·모션 부재로 화면이 정적이고 일관성이 없었다.

→ 외국인 근로자가 **불안한 법률 문서**를 확인하는 맥락이므로, 디자인 목표를 **신뢰감 + 가독성 + 앱다운 완성도**로 잡았다.

## 2. 디자인 방향 결정

- **톤:** Toss 스타일 — 넓은 여백, 큰 볼드 타이포, 단색 베이스 + 포인트 컬러 하나, 둥근 카드, 부드러운 모션.
- **포인트 컬러:** 토스블루 계열(`#3182F6`) 기준. `TODO: 최종 색상 팔레트는 다음 단계에서 조정(브랜드 색 확정).`
- **타깃 맞춤 변형:** 한국어가 익숙하지 않은 사용자를 위해 큰 글씨·고대비·명확한 정보 위계를 우선한다.

## 3. 화면별 결정

| 화면 | 핵심 결정 |
|---|---|
| 홈 | 큰 볼드 헤드라인 + 여백 + 종이 일러스트 + 하단 토스블루 CTA. 이모지 제거, 실제 아이콘 사용. |
| **촬영(카메라)** | **인앱 스캐너(`expo-camera`)로 실시간 뷰파인더 구현.** 계약서가 A4 세로 문서임을 반영해 뷰파인더를 **A4 비율(1:1.4)로 화면에 꽉 차게** 키우고, 네 모서리 가이드 + 스캔 라인 제공. 갤러리 선택도 병행. (안전장치: 막히면 네이티브 카메라 호출로 폴백) |
| 분석 결과 | 언어 세그먼트(한국어·English·Tiếng Việt) + "핵심 근로조건" 카드 + "확인이 필요해요" 카드(위험/확인/참고 3단계 색 구분) + 하단 참고용 고지. |

## 4. 보고서 어필포인트 (문장 그대로 사용 가능)

- **A4 비율 뷰파인더:** "실제 근로계약서가 A4 세로 문서인 점을 반영해, 촬영 뷰파인더를 A4 비율(1:1.4)에 맞추고 네 모서리 가이드를 제공함으로써 사용자가 문서를 정확히 담아 OCR 인식률을 높이도록 설계했다."
- **인앱 카메라 스캐너:** "단순 이미지 업로드가 아니라 앱 내 실시간 카메라 스캐너를 구현해, '촬영 → 분석 → 결과'라는 핵심 흐름이 앱 안에서 끊김 없이 이어지도록 했다."
- **다국어 결과 UI:** "한국어가 익숙하지 않은 외국인 근로자를 위해 분석 결과를 한국어·영어·베트남어로 즉시 전환할 수 있는 다국어 인터페이스를 제공한다."
- **위험도 색상 위계:** "위험·확인·참고 3단계 색상 구분으로 주의해야 할 조항을 한눈에 인지하도록 정보 위계를 설계했다."
- **자기결정 존중:** "법률 자문이 아닌 '이해 보조' 도구임을 모든 결과 화면에 명시해, 과도한 법적 판단을 피하고 사용자의 스스로 확인할 권리를 돕는다."
- **접근성 중심 모바일 UI:** "넓은 여백·큰 글씨·고대비의 모바일 UI로 가독성을 높여, 한국어·디지털 정보 접근성이 낮은 사용자도 쉽게 사용할 수 있도록 했다."

## 5. 적용 기술 (신청서 Part 2 · 사용 기술 / 프로토타입 설계용)

React Native + Expo / `expo-camera`(촬영) / `expo-image-picker`(갤러리) / `react-native-safe-area-context`(안전영역) / `react-native-reanimated`(모션) / `expo-symbols`(아이콘). 기존 프로젝트에 이미 설치된 라이브러리를 활용하므로 추가 도입 부담이 적다.
