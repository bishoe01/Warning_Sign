# OCR 출처 확인 UX 어필 포인트

> 작성일: 2026-06-05
> 관련: PRD F-17, roadmap Phase 10, `04_App/server/source_matching.py`, `04_App/mobile/src/app/result.tsx`

## 한 줄 요약

처음 결과 화면은 AI가 계약서 내용을 요약해 주지만, 사용자가 "이 설명이 실제 계약서 어디에서 나온 것인지" 확인하기 어려웠다. 그래서 결과 카드와 계약서 사진을 연결해, 사용자가 OCR로 읽힌 원문 위치를 직접 확인할 수 있는 UX로 개선했다.

## 왜 중요한가

근로계약서는 임금, 근로시간, 공제, 위약금처럼 민감한 조건을 담는다. 이런 문서에서 AI 설명만 보여주면 사용자는 다음을 알기 어렵다.

- AI가 실제 계약서 원문을 보고 말한 것인지
- OCR이 잘못 읽은 내용을 AI가 그대로 분석한 것인지
- 주의 조항이 계약서의 어느 부분에서 나온 것인지
- 사용자가 직접 원문과 비교해 확인할 수 있는지

따라서 이 기능은 단순한 화면 장식이 아니라 **신뢰 UX**다. 사용자가 AI 결과를 그대로 믿게 만드는 대신, 사진 속 근거를 보고 스스로 확인할 수 있게 한다.

## 개선 전

결과 카드에는 `계약서 원문` 인용문만 있었다.

```text
임금 비정상 기재

계약서 원문
"시 분 ~ 시 분"

설명...
```

하지만 사용자는 이 문장이 사진 어디에 있는지 다시 찾아야 했다. 특히 계약서가 여러 장이거나 OCR이 빈칸/표/괄호를 이상하게 읽으면, 결과가 실제 문서에서 나온 느낌이 약했다.

## 개선 후

결과 카드에 `사진에서 확인` 액션을 추가했다.

```text
확인이 필요한 조항 카드
→ 사진에서 확인
→ 원본 계약서 사진 열림
→ OCR로 읽은 위치 하이라이트
→ 하단 패널에 OCR 원문 + AI 설명 표시
```

매칭이 불확실하면 억지로 위치를 보여주지 않고 `원문 위치를 정확히 찾기 어려워요`라고 안내한다. 틀린 하이라이트를 보여주는 것보다, 근거를 찾지 못했다고 솔직히 말하는 편이 더 안전하다.

## 기술적으로 바꾼 점

### 1. OCR 줄 위치 보존

Google Vision OCR 결과에서 텍스트만 쓰지 않고, 줄 단위 bounding box를 정규화 좌표로 보존한다.

예:

```text
[p1-r4] 소정근로시간: 시 분 ~ 시 분
[p1-r5] 임금: 월 통상임금( )원
```

### 2. AI 응답에 근거 줄 ID 추가

처음에는 AI가 만든 `originalText` 문장을 OCR 줄에서 다시 문자열 검색했다. 하지만 AI가 `시 분 ~ 시 분`처럼 원문을 일부 재구성하면 위치를 찾지 못했다.

그래서 OCR 줄마다 `p1-r4` 같은 ID를 붙이고, AI가 각 주의 조항에 `sourceRegionIds`를 반환하도록 바꿨다.

```json
{
  "originalText": "소정근로시간: 시 분 ~ 시 분",
  "sourceRegionIds": ["p1-r4"]
}
```

서버는 이 ID를 기준으로 `cautionItems[].source` 좌표를 붙인다. ID 매칭이 실패할 때만 기존 문자열 매칭을 보조 수단으로 사용한다.

### 3. 모바일 이미지 뷰어 하이라이트

앱은 원본 이미지를 `contain` 방식으로 보여주기 때문에 화면 여백이 생긴다. 단순히 좌표를 화면 크기에 곱하면 박스가 어긋난다. 그래서 이미지가 실제로 렌더링된 영역을 계산한 뒤, 그 안에서 하이라이트 박스를 그린다.

추가로 사진 뷰어의 닫기 버튼이 상태바와 겹치던 문제를 safe area 기준으로 고쳤고, 사진 영역을 탭해도 닫히게 했다.

## 공모전에서 말할 수 있는 포인트

- 단순 OCR 요약이 아니라, **AI 설명의 근거를 원본 사진과 연결**했다.
- 사용자가 AI 결과를 맹신하지 않고 **계약서 원문에서 직접 확인**할 수 있다.
- 좌표를 AI가 상상해서 만들지 않고, **OCR bounding box와 서버 매칭 로직**으로 계산한다.
- 매칭이 불확실하면 하이라이트를 억지로 표시하지 않고, 낮은 신뢰도 안내를 보여준다.
- 민감한 계약서 이미지와 분석 결과는 서버 DB에 저장하지 않고, 앱 기기에만 저장한다.

## 발표 문장 초안

> 사용자가 AI 분석 결과를 볼 때 "이게 실제 계약서 어디에서 나온 내용인지" 확인하기 어렵다는 문제가 있었습니다. 그래서 OCR이 읽은 원문 위치를 계약서 사진 위에 하이라이트하고, 결과 카드에서 바로 `사진에서 확인`할 수 있게 개선했습니다. 이 기능은 AI를 믿으라고 강요하는 것이 아니라, 사용자가 원문을 직접 확인하도록 돕는 신뢰 UX입니다.

## 아직 확인해야 할 것

- 실제 iPhone Expo Go에서 새로 촬영한 계약서 사진으로 `sources=N/M` 로그가 0이 아닌지 확인한다.
- `사진에서 확인`이 올바른 페이지와 줄 근처를 가리키는지 확인한다.
- 박스가 줄보다 너무 크거나 어긋나면 Google Vision line/word grouping을 더 세밀하게 조정한다.
- 기존 저장 기록에는 새 source 정보가 없으므로, 새로 분석한 기록으로 QA한다.
