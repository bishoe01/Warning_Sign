# 기록 제목 UX 개선

## 문제

내 기록 목록이 분석 결과의 계약기간과 임금을 사실상 제목처럼 보여 주고 있었다. AI가 값이 없다고 판단한 항목은 `기재 없음`으로 내려오기 때문에, 사용자는 기록을 다시 열 때 어떤 계약서인지 구분하기 어렵다.

## 결정

- 분석 응답 스키마에는 제목 필드를 추가하지 않는다.
- 앱의 저장 기록 모델에만 사용자가 수정할 수 있는 `title`을 선택 필드로 추가한다.
- 촬영 검토 화면에서 분석 전에 기록 이름을 입력할 수 있게 한다.
- 사용자가 이름을 비워 두면 앱이 기존 분석 결과와 OCR 첫 줄을 기준으로 자동 제목을 만든다.
- 저장된 기록을 결과 화면에서 다시 열면 언어 선택 아래에 기록 이름 입력 영역을 보여 주고, 사용자가 수정할 수 있게 한다.

## 구현 범위

- `historyStore`에 `title`, `updateTitle`, `displayTitle` 추가
- `historyTitle` 순수 헬퍼 추가
- 촬영 검토 화면의 기록 이름 입력
- 로딩 저장 흐름의 제목 전달
- 홈 최근 기록과 내 기록 목록의 제목 표시
- 결과 상세 화면의 기록 제목 수정 UI

## 안전장치

- 기존 AsyncStorage 기록에는 `title`이 없을 수 있으므로 선택 필드로 유지한다.
- `기재 없음`, `<기재 없음>`, `Not written` 같은 값은 자동 제목 후보에서 제외한다.
- 사용자가 빈 제목을 저장하면 빈 문자열을 저장하지 않고 자동 제목으로 되돌린다.
