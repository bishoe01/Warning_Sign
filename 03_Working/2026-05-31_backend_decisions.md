> **확정(2026-05-31): 사용자가 "백엔드 유지" 선택.**
> 이유 = OCR/OpenAI 키를 앱에 넣으면 APK 에서 추출·도용 위험 → 키는 서버에만 둔다. 공모전 'API 서버 주소' 제출 가산도 고려.
> 백엔드는 **저장 목적이 아니다**(MVP 는 저장 없음, 추후 저장도 온디바이스로 충분). 백엔드의 유일한 역할 = **키 보관 + OCR/AI 호출 대행**.

## 핵심 설계: "키 없으면 샘플, 키 있으면 실제" 폴백

- 서버는 환경변수에 OCR/OpenAI 키가 **없으면 자동으로 샘플 응답**(sample mode), **있으면 실제 OCR+AI** 실행.
- 효과: 지금 당장 서버·앱이 끝까지 동작(데모 가능) + 키만 넣으면 **코드 변경 없이** 실제 파이프라인 전환.
- idea_.md 15절 "비상용 샘플 모드" + "실제 OCR/AI 동작" 목표를 동시에 충족.
- 앱도 서버 연결 실패 시 로컬 샘플로 폴백 → 발표장에서 안 깨진다.

## OCR 제공자: 기본 Google Vision (REST, API 키)

- 이유: 초보 팀이 키 발급~연결까지 **가장 빨리 도달**. 인쇄된 한국어 계약서 정확도 충분.
- idea_review는 CLOVA를 1순위(한국어 품질)로 봤음. CLOVA로 바꾸려면 `ocr_service.py` 한 곳만 교체 — 말해주면 전환.
- 환경변수 `GOOGLE_VISION_API_KEY` 없으면 샘플 OCR 텍스트 사용.

## AI: OpenAI Chat Completions, JSON 모드, 기본 gpt-4o-mini

- 이유: 구조화 추출에 충분 + 저렴(학생 팀). `OPENAI_MODEL`로 변경 가능.
- 프롬프트는 idea_.md 12절 기반. 응답은 pydantic으로 검증, 깨지면 샘플 폴백 + 로그.

## 안전

- 키는 서버 `.env`에만(앱에 안 넣음). 이미지/원문 미저장, 로그에 계약서 원문 안 남김.
- 단정 표현(위험/불법/무효/확정) 금지 톤 유지.

## 내가 검증한 것 / 못 한 것

- ✅ 샘플 모드 end-to-end(키 없이) — 서버 기동, `/health`, `/analyze-contract` 스키마까지 테스트.
- ⏳ 실제 Vision/OpenAI 호출 — 키가 없어 런타임 검증 불가. 키 넣고 1회 확인 필요(코드는 표준 API 형식대로 작성).

## 되돌리는 법

- OCR을 CLOVA로: "CLOVA로 바꿔줘" → `ocr_service.py` 교체.
- AI 모델 변경: `.env`의 `OPENAI_MODEL`.
- 샘플 강제(발표용): `.env`에서 키를 비우면 자동 샘플 모드.
