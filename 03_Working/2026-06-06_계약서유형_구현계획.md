> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 분석 전 사용자가 계약서 양식 2종 중 하나를 선택하고, 서버 AI가 선택된 양식에 맞는 체크리스트를 프롬프트에 적용하게 만든다.

**Architecture:** `contractType`은 계약서 양식/업종 축이고, `language`는 설명 언어 축이다. 1차 구현은 응답 스키마와 저장 기록 모델을 바꾸지 않고, 모바일 세션과 API payload, 서버 프롬프트 컨텍스트까지만 연결한다.

**Tech Stack:** React Native Expo, TypeScript, FastAPI, Pydantic, OpenAI Chat Completions JSON mode, unittest.

---

## 0. 구현 전 확인

- 네팔 PDF는 대표 양식 샘플이다. 네팔어 전용 기능으로 좁히지 않는다.
- 설명 언어는 기존 16개를 유지한다.
- `contractType`은 다음 두 값만 허용한다.

```text
manufacturing_construction_service
agriculture_livestock_fishery
```

- 1차 구현에서는 `AnalysisResult`에 `contractType`을 저장하지 않는다.
- 사용자가 잘못 선택해도 분석을 막지 않는다. OCR 제목과 선택값이 다를 수 있음을 안내하는 방향으로 둔다.

```ts
export const CONTRACT_TYPES = [
  {
    value: 'manufacturing_construction_service',
    labelKey: 'manufacturingConstructionService',
  },
  {
    value: 'agriculture_livestock_fishery',
    labelKey: 'agricultureLivestockFishery',
  },
] as const;

export type ContractType = (typeof CONTRACT_TYPES)[number]['value'];

export const DEFAULT_CONTRACT_TYPE: ContractType = 'manufacturing_construction_service';

export function isContractType(value: string | null | undefined): value is ContractType {
  return CONTRACT_TYPES.some((option) => option.value === value);
}

export function normalizeContractType(value: string | null | undefined): ContractType {
  return isContractType(value) ? value : DEFAULT_CONTRACT_TYPE;
}
```
