# Camera Capture UX 개편 spec (Phase 8 폴리시)

> 상태: 설계 합의 (2026-06-02)
> 대상 파일: `04_App/mobile/src/app/select.tsx` (단일 파일)
> 관련: Phase 8(멀티페이지 캡처) 직후 폴리시. 사용자 피드백 2026-06-02.

## 배경 (고칠 문제)
- **① 화면 이탈 후에도 카메라가 계속 active.** expo-router Stack은 촬영 화면을 push해도 언마운트하지 않고 밑에 쌓아둬서 `CameraView`가 loading·result에서도 계속 동작(배터리·카메라 사용 표시등).
- **② 갤러리로 골라도 큰 화면이 라이브 카메라.** Task 4의 "항상 라이브" 설계 결과. 갤러리로 고른 뒤에도 라이브 카메라가 큰 화면을 차지해 어색.

## 목표
- 카메라는 **"촬영 중일 때 + 화면이 포커스됐을 때"만 active.** 화면 이탈/검토 중엔 해제.
- 장을 추가하면(촬영/갤러리) 큰 화면이 **추가한 장들의 캐러셀**로 바뀌어 한 장씩 확인.

## 설계: 촬영 화면 2모드 (촬영 ↔ 검토)
상태: 기존 `pages: string[]`에 `mode: 'capture' | 'review'` 추가. (`busy` 등 기존 상태 유지)

### 촬영 모드 (`mode === 'capture'`)
- 큰 프레임 = 라이브 `CameraView`(+ 스캔라인 + 코너 브래킷). `active = focused && mode === 'capture'`.
- 셔터 촬영 → `pages`에 push, **촬영 모드 유지**(연속 촬영 OK).
- 하단: 갤러리 버튼 · 셔터 · (장 ≥1이면) "검토 N장 ›" → 검토 모드 전환. 그동안 찍은 썸네일 스트립(현행 유지). 썸네일 탭 → 그 장으로 검토 모드 진입.
- 0장이면 이 모드로 시작.

### 검토 모드 (`mode === 'review'`)
- 큰 프레임 = `pages` 가로 **캐러셀/페이저**(paging 스크롤, 한 장씩, 이미지 contain). "2 / 3" 인디케이터, 현재 장 × 삭제.
- `CameraView` 미렌더(= active 아님).
- 하단: "＋ 촬영"(→ 촬영 모드) · "갤러리"(추가) · "분석하기 (N장)".
- **갤러리로 장을 추가하면(어느 모드든) → 검토 모드로 전환**(고른 사진이 바로 캐러셀로 보임 = ② 해소).

### 카메라 active 로직 (① 해결)
- 포커스 추적: `useFocusEffect`로 `focused` 상태 관리(expo-router가 `useIsFocused`를 직접 export하지 않으므로 focus effect의 진입/정리로 true/false 토글).
- `<CameraView active={focused && mode === 'capture'} … />`. loading/result로 이동 → focused=false → 카메라 해제.

## 엣지 / 세부 결정
- **권한 없음 + 갤러리 장 있음**: 검토 모드(캐러셀)만 노출. "＋촬영" 시 기존 권한 게이팅(현행 유지)으로 안내.
- **검토 모드에서 마지막 장까지 삭제 → 0장**: 자동으로 촬영 모드 복귀.
- **분석하기**: 장 ≥1이면 노출. 위치는 검토 모드 하단(주 경로). 촬영 모드에선 "검토 N장"으로 모아 검토 후 분석 유도.
- **기존 fullscreen preview 모달 제거**: 검토 모드 캐러셀이 큰 화면 검토를 대체. (모달/`preview` 상태 삭제 — 본 변경으로 생긴 미사용 정리)
- 캡처 순서 = 페이지 순서(현행). 재정렬은 범위 밖.
