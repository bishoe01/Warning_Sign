# Phase 9 마이페이지 / 앱 IA 방향 메모 (다음 세션 brainstorm 시작점)

> 관련: roadmap Phase 9 / progress M7, PRD F-13·F-14·F-15
> 작성 이유: 사용자가 "이거 잘 정리해서 새 세션에서 하고 싶다"고 요청 → 다음 세션이 바로 brainstorm 착수할 수 있게 맥락 보존.

> 후속 결정(2026-06-02): 이 문서는 brainstorm 시작점으로만 본다. 기준 설계는 `2026-06-02_phase9_IA_다국어_설계.md`. 특히 "국가/국적 선택"이 아니라 **언어 토글/리스트 선택**으로 확정했다. 선택 언어는 앱 전체 UI와 분석 결과에 함께 적용한다.

## 배경 (사용자 피드백 2026-06-02)
- "내 기록" 기능 자체는 좋지만, **지금 구조는 너무 일회용/지속 안 되는 사이트처럼 보인다.**
- 해법 방향(둘 중 하나 또는 혼합):
  1. **마이페이지**를 둬서 "집/프로필" 영속 구조를 준다.
  2. 지금처럼 가되 **첫 화면(홈)을 더 구분**해서 기록 진입을 분명히 한다.
- 추가로 **읽고 싶은 언어 설정**도 원함 → 이미 Phase 9/M7(값 다국어 + 마이페이지 언어 설정).

## 수렴점
③(일회용 탈피) + ④(언어 설정)는 **마이페이지 하나로 수렴**한다:
- 마이페이지 = 허브: **내 기록**(기존 history) + **언어 설정** + **앱/프라이버시 안내**(로컬 전용 강조 = PRD §9 신뢰 포인트) + 앱에 영속 구조.
- 홈은 "분석 시작" 중심을 유지하되 마이페이지/기록 진입을 명확히.

## 새 세션에서 brainstorm할 것 (열린 질문)
1. **IA 택1**: (a) 하단 탭바(홈 / 기록 / 마이) · (b) 홈 + 우상단 마이페이지 진입 · (c) 현행 유지 + 홈 구분만 강화. — 외국인 근로자 대상 단순성 고려.
2. **마이페이지 구성**: 언어 선택(앱 전체 UI + 결과 언어), 내 기록 진입, 앱 정보·프라이버시(로컬 저장) 안내.
3. **Phase 9 다국어와 통합**: 선택 언어값 → result 화면의 **값까지** 선택 언어. 원문(originalText)은 한국어 유지 여부. AI 직접 생성 vs 번역. (M7-T1~T4, PRD F-14·F-15)
4. **기존 진입점 흡수 여부**: 현재 홈 우상단 "내 기록"(Phase 8)을 마이페이지로 옮길지, 둘 다 둘지.

## 현재 상태 참고 (Phase 8 결과)
- 홈(`index.tsx`) 우상단에 "내 기록" 칩 → `/history`.
- `/history`: 목록(썸네일·N장·날짜·요약·주의건수·샘플배지), 스와이프/전체 삭제, "이 기기에만 저장" 안내.
- `result.tsx`: 새 분석 시 "이 기기에 저장됐어요" 안내.
- 다국어는 현재 라벨만 ko/en/vi(`labels`, `LanguageTabs`). 값은 아직 한국어(서버 응답).

## UX 검수 결과 (2026-06-02) — Phase 9/마이페이지로 흡수
대상 사용자(한국어 서툰 외국인 근로자) 렌즈 검수에서 나온 항목:
- 🔴 **앱 "껍데기"가 전부 한국어.** 네비/버튼/힌트(분석하기·내 기록·검토·전체 삭제·로딩 문구 등)가 한국어 전용. 콘텐츠는 ko/en/vi 탭이 있으나 UI 자체가 한국어 → 한국어 못 읽는 대상이 **버튼을 못 읽는 모순.** **Phase 9에서 "콘텐츠 값 번역"보다 "앱 전역 언어 전환"을 1순위로** 잡는 것을 추천.
- 🔴 **결과 "값" 미번역.** 영/베 탭에서도 요약 값("월 2,060,740원")·주의 조항 **제목**은 한국어(설명문은 번역됨). 제목 미번역은 빠지기 쉬운 디테일.
- 🟢 **결과 진입 시 매번 'ko' 탭으로 시작** → 언어 설정으로 **기본 결과 언어 기억**하면 자연 해결.

폴리시 백로그(언어와 별개 — Phase 9 또는 별도 plan에 메모):
- 목록 개별 삭제가 **스와이프 전용 → 발견성 약함**(상세엔 보이는 휴지통 있음). 행 affordance 보강 고려.
- **로딩 단계가 타이머(900ms)라 실제 진행과 무관** → 멀티페이지 real OCR 길면 멈춘 듯 보임.
- 목록 정보 밀도/작은 폰트 → 대상 사용자엔 더 큰 터치/폰트.
