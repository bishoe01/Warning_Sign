# 외국인 근로자를 위한 AI 근로계약서 도우미 MVP 기획서

## 1. 프로젝트 한 줄 요약

**외국인 근로자가 스마트폰으로 한국어 근로계약서를 촬영하면, 실제 OCR로 텍스트를 추출하고 AI가 핵심 근로조건과 확인이 필요한 조항을 쉬운 한국어 및 모국어로 설명해주는 모바일 앱 MVP**를 만든다.

> 이 MVP의 기준은 “그럴듯한 목업”이 아니라, 최소 기능이라도 실제로 개발되어 돌아가는 프로토타입이다. 카메라 촬영, OCR 텍스트 추출, AI 분석, 결과 표시까지 하나의 흐름으로 연결하는 것을 목표로 한다.

## 2. 최종 개발 방향

이번 공모전용 MVP는 **React Native + Expo 기반 Android 앱**으로 개발한다. 웹앱은 실행은 쉽지만 카메라 기반 서비스라는 느낌이 약하고, Flutter는 개발 경험이 거의 없는 팀에게 학습 부담이 크다. 따라서 Expo를 사용해 모바일 앱 형태를 빠르게 만들고, 실제 OCR과 AI 분석은 간단한 백엔드 API를 통해 처리한다.

| 항목 | 최종 선택 | 이유 |
|---|---|---|
| 앱 프레임워크 | **React Native + Expo** | 개발 초보 팀이 카메라, 이미지 선택, 화면 전환을 비교적 빠르게 구현할 수 있다. |
| 앱 언어 | **TypeScript** | 코드 오류를 줄이고 화면 데이터 구조를 명확하게 관리할 수 있다. |
| 백엔드 | **FastAPI** | OCR API와 AI API 키를 안전하게 숨기고, 이미지 분석 요청을 처리하기 좋다. |
| OCR | **Google Vision API 또는 Naver CLOVA OCR** | 한국어 계약서 이미지를 실제 텍스트로 변환하기 위한 핵심 기능이다. |
| AI 분석 | **OpenAI API** | OCR 결과를 기반으로 핵심 근로조건 요약, 확인 필요 조항 설명, 번역을 수행한다. |
| DB | **MVP에서는 사용하지 않음** | 로그인과 저장 기능 없이도 핵심 데모가 가능하다. |
| 배포 | **Android APK + 데모 영상** | 공모전에서 실제 앱처럼 보여주고, 설치가 어려운 경우 영상으로 흐름을 확인시킬 수 있다. |

## 3. 문제 정의

한국에서 일하는 외국인 근로자는 근로계약서를 작성할 때 한국어 문장을 정확히 이해하기 어렵다. 특히 임금, 근로시간, 휴일, 계약기간, 수습기간, 숙식비 공제, 위약금, 퇴직금 같은 내용은 실제 권리와 생활에 직접적인 영향을 준다. 하지만 계약서 문장은 어렵고, 일반 번역기만으로는 조항의 의미나 주의할 점을 충분히 파악하기 어렵다.

이 서비스는 외국인 근로자가 계약서를 받은 순간 스마트폰으로 촬영하여, **계약서의 핵심 조건과 주의해야 할 조항을 바로 확인**할 수 있도록 돕는다.

## 4. 핵심 사용자 시나리오

사용자는 앱을 실행하고 계약서를 촬영한다. 앱은 이미지를 서버로 전송하고, 서버는 OCR을 통해 계약서의 한국어 텍스트를 추출한다. 이후 AI는 추출된 텍스트에서 임금, 근무시간, 휴일, 계약기간, 공제 항목, 확인이 필요한 조항을 분석한다. 앱은 분석 결과를 카드 형태로 보여주고, 사용자가 언어를 선택하면 쉬운 한국어, 영어, 베트남어 설명으로 내용을 확인할 수 있게 한다.

```text
앱 실행
  ↓
계약서 촬영 또는 이미지 선택
  ↓
이미지 서버 업로드
  ↓
OCR API로 계약서 텍스트 추출
  ↓
AI가 핵심 근로조건과 확인 필요 조항 분석
  ↓
앱에서 결과 카드 표시
  ↓
언어 선택 후 쉬운 설명 확인
```

## 5. MVP에서 반드시 구현할 기능

이번 MVP에서는 기능을 넓히기보다, **촬영부터 분석 결과까지 실제로 연결되는 흐름**을 만드는 것이 중요하다. 따라서 아래 기능은 필수로 구현한다.

| 우선순위 | 기능 | 설명 | 완료 기준 |
|---|---|---|---|
| 1 | 홈 화면 | 서비스 이름과 시작 버튼을 보여준다. | 앱 실행 시 첫 화면이 보인다. |
| 1 | 계약서 촬영/이미지 선택 | 사용자가 계약서 사진을 찍거나 갤러리에서 선택한다. | 이미지가 앱 화면에 표시된다. |
| 1 | 이미지 업로드 | 선택한 이미지를 백엔드 API로 전송한다. | 서버가 이미지를 정상적으로 받는다. |
| 1 | OCR 텍스트 추출 | OCR API를 통해 계약서 텍스트를 추출한다. | OCR 결과 텍스트가 서버에서 반환된다. |
| 1 | AI 분석 | OCR 텍스트를 AI에 보내 요약과 확인 필요 조항을 생성한다. | JSON 형태 분석 결과가 반환된다. |
| 1 | 분석 결과 화면 | 핵심 근로조건과 확인 필요 조항을 카드로 보여준다. | 앱에서 결과가 읽기 쉽게 표시된다. |
| 2 | 다국어 설명 | 한국어, 영어, 베트남어 설명을 전환한다. | 탭을 누르면 언어가 바뀐다. |
| 2 | 법률 자문 아님 고지 | 결과가 참고용임을 명확히 표시한다. | 결과 화면 하단에 고지가 보인다. |

## 6. MVP에서 제외할 기능

개발 경험이 거의 없는 팀이므로 범위를 과도하게 넓히면 완성 가능성이 떨어진다. 아래 기능은 좋은 기능이지만 이번 제출용 MVP에서는 제외한다.

| 제외 기능 | 제외 이유 |
|---|---|
| 회원가입·로그인 | 핵심 기능 구현에 필요하지 않고 개발 부담이 크다. |
| 계약서 저장함 | DB와 개인정보 관리가 필요해 범위가 커진다. |
| 상담 예약 | 외부 기관 연동이 필요해 MVP 범위를 벗어난다. |
| 관리자 페이지 | 공모전 데모에는 필수적이지 않다. |
| iOS 배포 | TestFlight와 애플 개발자 계정 이슈가 있어 초보 팀에게 부담이 크다. |
| 완전한 법률 판단 | 법적 책임 문제가 있으므로 “확인 필요 안내” 수준으로 제한한다. |

## 7. 시스템 구조

API 키를 모바일 앱에 직접 넣으면 노출 위험이 있다. 따라서 OCR과 AI 호출은 앱에서 직접 하지 않고, **FastAPI 백엔드**에서 처리한다. 앱은 이미지를 서버로 보내고, 서버가 OCR과 AI 분석을 수행한 뒤 결과 JSON만 앱에 반환한다.

```text
[Expo Mobile App]
  └─ 계약서 이미지 촬영/선택
      ↓ multipart/form-data 업로드
[FastAPI Backend]
  └─ 이미지 수신
  └─ OCR API 호출
  └─ OCR 텍스트 정리
  └─ OpenAI API 호출
      ↓ JSON 결과 반환
[Expo Mobile App]
  └─ 요약 카드 표시
  └─ 확인 필요 조항 카드 표시
  └─ 언어별 설명 표시
```

## 8. 추천 API 설계

백엔드는 MVP 기준으로 API 하나만 있어도 된다. 앱에서 이미지를 업로드하면 서버가 OCR과 AI 분석을 한 번에 처리해서 결과를 반환한다.

| 메서드 | 경로 | 설명 |
|---|---|---|
| `POST` | `/analyze-contract` | 계약서 이미지를 업로드하면 OCR과 AI 분석 결과를 반환한다. |
| `GET` | `/health` | 서버가 정상 작동하는지 확인한다. |

### 8.1 요청 형식

```http
POST /analyze-contract
Content-Type: multipart/form-data

file: contract_image.jpg
language: ko | en | vi
```

### 8.2 응답 형식

```json
{
  "ocrText": "근로계약서 OCR 원문 텍스트",
  "summary": {
    "salary": "월 2,060,740원",
    "workHours": "주 5일, 1일 8시간",
    "holiday": "매주 일요일",
    "contractPeriod": "2026.06.01 ~ 2027.05.31",
    "deduction": "숙식비 월 200,000원 공제"
  },
  "cautionItems": [
    {
      "level": "check",
      "title": "퇴사 시 위약금 조항",
      "originalText": "근로자가 1년 이내 퇴사할 경우 교육비 100만원을 반환한다.",
      "explanationKo": "퇴사할 때 돈을 내야 한다는 내용입니다. 문제가 될 수 있으니 상담이 필요합니다.",
      "explanationEn": "This clause says you may have to pay money if you quit early. You should get advice before signing.",
      "explanationVi": "Điều khoản này nói rằng bạn có thể phải trả tiền nếu nghỉ việc sớm. Bạn nên được tư vấn trước khi ký."
    }
  ],
  "notice": "이 분석은 법률 자문이 아니라 참고용 안내입니다. 중요한 계약은 전문가 상담을 권장합니다."
}
```

## 9. 화면 구성

앱 화면은 최소한으로 구성한다. 공모전에서는 화면이 많을수록 좋아 보이는 것이 아니라, 핵심 흐름이 끊기지 않는 것이 중요하다.

| 화면 | 설명 | 필수 여부 |
|---|---|---|
| 홈 화면 | 서비스 소개와 “계약서 분석 시작” 버튼을 보여준다. | 필수 |
| 촬영/선택 화면 | 카메라 촬영 또는 갤러리 이미지 선택을 제공한다. | 필수 |
| 분석 중 화면 | OCR과 AI 분석 진행 상태를 단계별로 보여준다. | 필수 |
| 결과 화면 | 핵심 근로조건 요약 카드와 확인 필요 조항 카드를 보여준다. | 필수 |
| 언어 전환 UI | 한국어, 영어, 베트남어 설명을 전환한다. | 필수 |
| OCR 원문 보기 | OCR로 추출된 원문을 접어서 볼 수 있게 한다. | 선택 |

## 10. 분석 중 화면 문구

실제로 OCR과 AI 요청이 몇 초 걸릴 수 있으므로, 사용자가 기다리는 동안 단계별 메시지를 보여준다.

| 단계 | 표시 문구 |
|---|---|
| 1단계 | 계약서 이미지를 업로드하고 있습니다. |
| 2단계 | OCR로 계약서 문장을 읽고 있습니다. |
| 3단계 | AI가 임금, 근무시간, 휴일 조건을 분석하고 있습니다. |
| 4단계 | 확인이 필요한 조항을 쉬운 문장으로 정리하고 있습니다. |

## 11. AI 분석 기준

AI는 계약서를 “법적으로 확정 판단”하는 것이 아니라, 사용자가 이해하기 쉽게 구조화하고 주의할 부분을 알려주는 역할을 한다. 확인 필요 조항은 확정적인 표현 대신 “확인이 필요합니다”, “상담을 권장합니다”처럼 안내형 문장으로 표현한다.

| 분석 항목 | 설명 |
|---|---|
| 임금 | 월급, 시급, 공제 항목, 실제 수령액에 영향을 주는 내용을 찾는다. |
| 근무시간 | 하루 근무시간, 주 근무일, 연장근로 가능성을 확인한다. |
| 휴일 | 주휴일, 휴게시간, 휴가 관련 내용을 확인한다. |
| 계약기간 | 시작일, 종료일, 수습기간을 정리한다. |
| 확인 필요 조항 | 위약금, 과도한 공제, 불명확한 연장근로, 임금 미기재 등을 표시한다. |
| 쉬운 설명 | 어려운 계약 문장을 짧고 쉬운 문장으로 바꾼다. |
| 번역 | 영어와 베트남어로 핵심 설명을 제공한다. |

## 12. 프롬프트 초안

백엔드에서 OCR 텍스트를 AI에 보낼 때는 아래와 같은 방식으로 요청한다. AI 응답은 앱에서 바로 사용하기 쉽도록 반드시 JSON 형태로 받는다.

```text
너는 외국인 근로자를 돕는 근로계약서 설명 도우미다.
아래 OCR 텍스트는 한국어 근로계약서에서 추출한 내용이다.

해야 할 일:
1. 임금, 근무시간, 휴일, 계약기간, 공제 항목을 추출한다.
2. 외국인 근로자가 확인해야 할 주의 조항을 찾는다.
3. 확인 단계는 check, review, info 중 하나로 분류한다.
4. 설명은 쉬운 한국어, 영어, 베트남어로 작성한다.
5. 법률 자문이 아니라 참고용 안내라는 문구를 포함한다.
6. 응답은 반드시 JSON으로만 작성한다.

OCR 텍스트:
{ocr_text}
```

## 13. 개발 순서

개발 초보 팀은 아래 순서대로 진행한다. 핵심은 처음부터 OCR과 AI를 한 번에 붙이려 하지 말고, 화면 → 이미지 업로드 → OCR → AI 순서로 연결하는 것이다.

| 순서 | 작업 | 목표 |
|---|---|---|
| 1 | Expo 프로젝트 생성 | 앱이 휴대폰 또는 에뮬레이터에서 실행된다. |
| 2 | 홈 화면 제작 | 시작 버튼이 보인다. |
| 3 | 이미지 선택 기능 구현 | 갤러리에서 계약서 이미지를 선택할 수 있다. |
| 4 | 카메라 촬영 기능 구현 | 카메라로 계약서 사진을 찍을 수 있다. |
| 5 | FastAPI 서버 생성 | `/health` API가 응답한다. |
| 6 | 이미지 업로드 연결 | 앱에서 서버로 이미지가 전송된다. |
| 7 | OCR API 연결 | 서버가 이미지에서 텍스트를 추출한다. |
| 8 | AI API 연결 | OCR 텍스트를 분석해서 JSON을 만든다. |
| 9 | 결과 화면 연결 | 서버 응답을 앱 카드 UI로 표시한다. |
| 10 | APK 빌드 | Android 설치 파일을 만든다. |
| 11 | 데모 영상 촬영 | 전체 흐름을 1분 안에 보여준다. |

## 14. 개발 중 막히면 줄이는 순서

실제 OCR과 AI 개발을 목표로 하되, 시간이 부족하면 기능을 줄이는 순서를 미리 정해둔다. 단, 기본 목표는 실제 OCR 연동이다.

| 상황 | 줄이는 방법 |
|---|---|
| 카메라 구현이 막힘 | 갤러리 이미지 선택만 먼저 구현한다. |
| OCR API 연결이 막힘 | OCR API 호출 부분만 집중해서 해결하고, AI는 임시 JSON으로 둔다. |
| AI JSON 파싱이 막힘 | AI 응답을 텍스트 설명으로 먼저 보여주고, 나중에 JSON 구조화한다. |
| APK 빌드가 막힘 | Expo 실행 영상과 개발 환경 실행 화면을 제출 보조자료로 준비한다. |

## 15. 샘플 모드의 위치

샘플 모드는 기본 개발 방향이 아니다. 다만 공모전 발표 당일 인터넷, API 키, OCR 장애가 생길 수 있으므로 **비상용 플랜 B**로만 둔다. 실제 개발과 제출의 기본 설명은 “실제 OCR과 AI 분석이 동작하는 MVP”로 한다.

| 구분 | 방향 |
|---|---|
| 기본 개발 | 실제 OCR API와 AI API를 연결한다. |
| 기본 제출 설명 | 계약서 이미지를 실제로 OCR 처리하고 AI가 분석한다고 설명한다. |
| 비상용 샘플 모드 | 발표 당일 장애 대비용으로만 준비한다. |
| 화면 노출 | 가능하면 일반 사용 흐름에서는 샘플 모드를 숨긴다. |

## 16. 제출물 구성

공모전 제출은 앱만 던지는 것이 아니라, 심사자가 이해하기 쉬운 자료를 함께 낸다.

| 제출물 | 필수 여부 | 설명 |
|---|---|---|
| Android APK | 필수 | 실제 앱 설치 파일이다. |
| 데모 영상 | 필수 | 앱 설치 없이도 전체 흐름을 볼 수 있다. |
| 발표자료 | 필수 | 문제 정의, 기술 구조, 차별점, 기대효과를 설명한다. |
| GitHub 링크 | 권장 | 실제 개발 결과물임을 보여준다. |
| API 서버 주소 | 권장 | 심사자가 실제 분석 흐름을 확인할 수 있다. |

## 17. 발표 포인트

발표에서는 “번역 앱”이 아니라 **AI 근로계약 확인 도우미**라는 점을 강조한다. 단순히 한국어를 다른 언어로 바꾸는 서비스가 아니라, 계약서에서 근로조건을 뽑고 주의해야 할 조항을 설명해주는 서비스라는 점이 차별점이다.

| 발표 항목 | 핵심 메시지 |
|---|---|
| 문제 | 외국인 근로자는 한국어 계약서를 이해하기 어렵고, 불리한 조항을 놓칠 수 있다. |
| 해결 | 계약서를 촬영하면 OCR과 AI가 핵심 조건과 확인 필요 조항을 설명한다. |
| 기술 | 모바일 카메라, OCR, LLM 요약, 확인 필요 조항 분류, 다국어 설명을 결합했다. |
| 차별점 | 단순 번역이 아니라 근로계약의 중요한 항목을 구조화해서 보여준다. |
| 기대효과 | 계약 전 정보 격차를 줄이고 외국인 근로자의 권리 보호에 도움을 준다. |
| 확장성 | 표준근로계약서 비교, 상담기관 연결, 계약서 저장함으로 확장할 수 있다. |

## 20. 최종 결론

이 MVP는 목업이나 샘플 결과만 보여주는 앱이 아니라, **카메라로 계약서를 촬영하고 실제 OCR과 AI 분석을 거쳐 결과를 보여주는 최소 기능 제품**을 목표로 한다. 다만 개발 초보 팀이라는 현실을 고려해 로그인, DB, 상담 예약, iOS 배포 같은 부가 기능은 제외한다.

최종 결정은 다음과 같다.

> **React Native + Expo로 Android 앱을 만들고, FastAPI 백엔드에서 OCR API와 OpenAI API를 연결한다. 앱은 계약서 촬영, 이미지 업로드, OCR/AI 분석 결과 표시까지 구현한다. 제출은 Android APK와 데모 영상을 함께 준비한다.**
