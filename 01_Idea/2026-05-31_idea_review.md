# 2026-05-31 idea_.md 점검

## 결론

`01_Idea/idea_.md`는 "외국인 근로자 대상 AI 근로계약서 도우미"로 개발 방향이 꽤 구체적이다. Expo 앱, FastAPI 백엔드, OCR, AI 분석, 결과 카드까지 MVP 흐름이 선명해서 바로 개발 계획으로 옮길 수 있다.

다만 기존 작업공간의 기준이었던 "경계선 지능 청소년·청년" 방향과 대상 사용자가 다르다. 지금 개발을 시작하려면 **외국인 근로자 방향으로 고정할지, 이전 방향으로 되돌릴지**를 먼저 문서상으로 분리해야 한다. 이번 정리는 `idea_.md`를 현재 개발 기준안으로 보고 점검했다.

## 좋은 점

- "그럴듯한 목업"이 아니라 촬영, OCR, AI 분석, 결과 표시까지 실제 동작을 목표로 한다.
- 모바일 앱과 백엔드를 분리해 API 키 노출 문제를 피하려는 구조가 맞다.
- 회원가입, DB, 상담 예약, iOS 배포를 제외해 MVP 범위를 잘 줄였다.
- 임금, 근로시간, 휴일, 계약기간, 공제, 위약금 등 근로계약서의 핵심 확인 항목이 드러난다.
- 법률 판단이 아니라 참고용 안내라는 제한을 문서에 넣었다.

## 개발 기준으로 정리한 MVP

### 사용자

한국어 근로계약서를 혼자 확인해야 하는 외국인 근로자. 특히 임금, 근로시간, 공제, 수습기간, 위약금 같은 조항을 놓치기 쉬운 사용자를 우선한다.

### 핵심 가치

계약서를 대신 판단하지 않고, 사용자가 계약서의 중요한 내용을 이해한 뒤 스스로 질문하거나 상담을 받을 수 있게 돕는다.

### 필수 흐름

```text
앱 실행
→ 계약서 촬영 또는 이미지 선택
→ 이미지 업로드
→ OCR 텍스트 추출
→ AI가 핵심 근로조건과 주의 조항을 구조화
→ 결과 카드 표시
→ 언어 전환
→ 참고용 안내 표시
```

### 첫 개발 범위

- Expo Android 앱
- 홈 화면
- 이미지 선택 우선, 카메라 촬영은 그 다음
- FastAPI `/health`
- FastAPI `/analyze-contract`
- OCR API 1개 연결
- OpenAI API 분석 결과 JSON 생성
- 결과 카드 UI
- 법률 자문 아님 고지
- 샘플 계약서/샘플 응답 fallback

## 신청서에서 조심할 표현

| 피할 표현 | 바꿀 표현 |
|---|---|
| AI가 불법 계약을 판정한다 | AI가 확인이 필요한 표현을 알려준다 |
| 위험 조항을 탐지한다 | 주의해서 볼 조항을 분류한다 |
| 사용자를 보호한다 | 사용자가 스스로 이해하고 질문할 수 있게 돕는다 |
| 법률 상담을 제공한다 | 상담이 필요한 경우를 안내한다 |
| 계약서 문제를 해결한다 | 계약 전 정보 격차를 줄인다 |
